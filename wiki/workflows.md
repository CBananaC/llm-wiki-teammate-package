# Workflows

## Source-to-review workflow

1. Start from registered source evidence or the canonical Stage 1 dataset.
2. Select the relevant canonical skill from [[skills]].
3. Run the extraction, summary, or pairing task reproducibly.
4. Save generated candidates in a review bundle.
5. Review evidence and uncertainty in the formal or comparison tool.
6. Promote only accepted results into formal state or a concise wiki conclusion.
7. Record the coherent project change in `PROJECT_LOG.md`.

## Correction workflow

1. Identify whether the failure concerns evidence, schema, prompt, source data, or interpretation.
2. Correct derived material without overwriting raw evidence.
3. Update the canonical skill or rule when the problem can recur.
4. Rerun a small representative test.
5. Record verification and remaining uncertainty.

## Data boundaries

- Formal state and sample state must remain isolated.
- Review bundles are research evidence, not disposable build files.
- Browser-local edits do not become canonical until exported and deliberately reviewed.
- Generated data should not be copied into the wiki merely for documentation.
